# file_rename

[mv2title](https://github.com/Shannon-toppo/mv2title) ライブラリを利用して、音声ファイルの**曲名をメタデータ（タイトルタグ）に書き込む**ためのスクリプト集です。

YouTube などの MV はファイル名・タイトル（アーティスト名・`feat.`・`Official Music Video`・括弧類）をローカル LLM で曲名に整形し、ファイルのタグへ書き込みます。

## 構成

| ファイル | 役割 |
|:--|:--|
| `core.py` | 共通コア: DL・曲名推測・タグ書き込み（`.env` 読み込みもここ。print しない GUI 非依存層） |
| `rename.py` | `files/` 内の音声を走査し、ファイル名から曲名を推測してメタデータに書き込む |
| `download.py` | yt-dlp で URL（または再生リスト）から音声をDLし、チャンネル名をヒントにタグ付け |
| `gui/` | PySide6 製の GUI（テーブル UI で DL → 推定 → 書き込みを操作） |
| `files/` | 処理対象の音声を置く作業フォルダ（中身はバージョン管理対象外） |
| `list.txt` | `download.py -a` に渡す URL 一覧（各自のローカルファイル。git 管理外） |

> このリポジトリは隣接する `../mv2title/` パッケージに **editable なパス依存**（`pyproject.toml` の `[tool.uv.sources]`）で依存します。別環境で使う場合は、`mv2title` を同じ親フォルダ直下に clone してから `uv sync` してください。

## 必要なもの

- **Python >= 3.12** と **uv**
  ```bash
  uv sync   # 初回に一度。mutagen / yt-dlp / mv2title 入りの venv を作る
  ```
- **ffmpeg**（`download.py` の mp3 / wav 変換に必要。PATH を通しておくこと。Mac: `brew install ffmpeg`）
- **OpenAI 互換の LLM エンドポイント**（LM Studio / llama.cpp など）。設定は `../mv2title/.env` で行います（`BASE_URL` / `API_KEY` / `MODEL` / `SYSTEM_PROMPT`）。詳細は mv2title 側の README を参照。

> Mac で初めてセットアップする場合は、[docs/macos-setup.md](docs/macos-setup.md)（Homebrew の導入から起動までの手順書）を参照してください。

## 使い方

### 既存ファイルにタグ付け

`files/` に音声（`.mp3` / `.wav` / `.m4a`）を置いて実行します。

```bash
uv run python rename.py
```

ファイル名から曲名を推測し、各ファイルのタイトルタグに書き込みます。

### URL からダウンロードしてタグ付け

```bash
# 単一URL（形式の既定は mp3）
uv run python download.py "https://www.youtube.com/watch?v=XXXX"

# 形式を指定
uv run python download.py "https://www.youtube.com/watch?v=XXXX" -f wav

# テキストファイルから一括（1行1URL、空行と # 始まりの行は無視）
uv run python download.py -a list.txt -f mp3
```

| オプション | 既定 | 説明 |
|:--|:--:|:--|
| `url`（位置引数） | — | ダウンロードする動画 URL |
| `-a`, `--batch-file` | — | URL を1行ずつ記入したテキストファイル |
| `-f`, `--format` | `mp3` | 保存形式（`mp3` / `wav` / `m4a`） |

`url` と `-a` はどちらか一方を指定します。

#### 再生リストの扱い

- `playlist?list=...`（純粋な再生リスト）→ 含まれる全動画をDL
- `watch?v=...&list=...`（動画＋リスト混在）→ その**動画1本のみ**DL
- 再生リスト中の取得失敗（非公開・地域制限など）はスキップして続行します

### GUI で操作する

Excel 風のテーブルで、URL 貼り付け → ダウンロード → タイトル推定 → 書き込みを一気通貫で行えます。

```bash
uv run python -m gui
```

- **上段**: URL 入力欄（改行区切りで複数貼り付け可）＋ [追加] / [ファイル追加] / [files/ 取り込み]。対応拡張子のローカルファイルはテーブルへドラッグ&ドロップでも追加できます。
- **ツールバー**: [▶ 実行]（実行時点の全行を処理）/ [■ 停止] / 形式コンボ（mp3 / wav / m4a）/ ☑ 自動書き込み（既定 ON）。
- **テーブル**: 1 行 = 1 曲。列は `#` / 元タイトル / チャンネル / 推定タイトル / アーティスト / 状態 / 形式。状態別に背景色が付きます（完了=緑 / エラー=赤 / 要確認=黄 / 処理中=青。色付き行は文字色を濃色に固定するので OS のダークテーマでも読めます）。DL 中は状態列に**進捗バー**を描画し、再生リストなら「DL中 2/5 45%」のように**何番目のダウンロードか**も表示します。
- **手動修正**: 「推定タイトル」と「アーティスト」列が編集可。タイトルを編集した行は `✎` マークが付き、状態が確認待ちに戻り、再推定で上書きされません（[選択行を再推定] は明示操作なので手動行も上書きします）。
- **アーティスト欄**: 推定はしません。手動入力するか、[チャンネル名→アーティスト] ボタンで選択行（未選択なら全行）にチャンネル名をそのままコピーします。アーティストはタイトルと一緒にタグへ書き込まれます（mp3/wav=`TPE1`、m4a=`©ART`。空欄なら書き込みません）。書き込み済みの行のアーティストを編集すると「確認待ち」に戻るので、[選択行を書き込み] で反映してください。
- **下段**: [選択行を再推定] / [選択行を書き込み]（確認待ち行の確定用）/ [チャンネル名→アーティスト] / [行削除]（テーブルから外すだけ。ファイルは消しません）。書き込み後はステータスバーに「書き込み: 完了 N 件 / スキップ M 件 / 失敗 K 件」の集計が出ます。
- 自動書き込み OFF のときは推定後に「確認待ち」で停止し、内容を確認してから [選択行を書き込み] で確定できます。

#### Excel 風の操作

- **キーボード**: `F2` またはセル上で直接タイプで編集開始、`Enter` で編集確定して下のセルへ移動。`Delete`（`Backspace`）（Mac: `delete`）で選択行の推定タイトルをクリアし、再推定対象（キュー）に戻します。
- **コピー / ペースト（TSV、Excel と相互運用）**: `Ctrl+C`（Mac: `Cmd+C`）で選択セル範囲をタブ区切りでクリップボードへ（手動編集の `✎ ` プレフィックスは含まない素の値）。`Ctrl+V`（Mac: `Cmd+V`）は現在セルを左上として貼り付けますが、**編集可能な「推定タイトル」「アーティスト」列に落ちるセルのみ**反映します（他列は無視）。Excel の範囲を貼り付けて一括更新できます。
- **元に戻す / やり直し**: `Ctrl+Z` / `Ctrl+Y`（Mac: `Cmd+Z` / `Cmd+Shift+Z`）。タイトル編集・ペースト・`Delete` クリアが対象です（ペーストは 1 回の `Ctrl+Z` でまとめて戻ります。行削除の取り消しは対象外）。
- **右クリックメニュー**: [選択行を再推定] / [選択行を書き込み] / [行削除] / [URL をブラウザで開く]（URL を持つ行でのみ有効）。実行中は再推定・書き込み・行削除が無効になります。
- **ソート**: ヘッダクリックで列ソート（`#` 列は現在の並び順のまま）。列幅ドラッグも可能。パイプライン実行中はソートを無効化します。
- ウィンドウサイズ・列幅・自動書き込みチェック・形式コンボは終了時に保存し、次回起動時に復元します（`QSettings`）。

#### 設定・堅牢化

- **[設定] ダイアログ**: 保存先フォルダ / 既定の音声形式 / 推定バッチサイズ（1 回の LLM リクエストに含めるタイトル数）/ 自動書き込みの既定値 / **再生リスト付き動画 URL の扱い**（既定はその動画 1 本のみ。ON にすると `watch?v=...&list=...` でもリスト全体を展開してダウンロード）/ **テーマ**（システムに合わせる〔既定〕・ライト・ダーク。Qt 6.8+ の配色切替を使用）/ **ログレベル**（[ログ] パネルに表示する最小レベル。詳細（すべて）・情報・警告〔既定〕・エラーのみ）を変更できます（`QSettings` で永続化）。接続先（`BASE_URL` / `MODEL`）は読み取り専用表示で、変更は `../mv2title/.env` で行います。[接続テスト] でエンドポイントの疎通を確認できます。
- **縮退モード**: [実行] 時に LLM エンドポイントへ届かない場合、自動で **DL のみ**実行して行を「キュー」のまま残します（ステータスバーに通知）。サーバを起動してから再度 [実行] すれば推定から続きが処理されます。
- **[ログ] パネル**: 折りたたみ式のログ表示（既定は非表示）。mv2title のパースフォールバック warning に加え、**yt-dlp のダウンロードログ**も表示します（GUI 経由の DL は yt-dlp の出力を logging へ切り替えるため）。表示する最小レベルは [設定] のログレベルで変更できます（既定は警告）。「詳細（すべて）」にすると yt-dlp の進捗行も流れるため流量がかなり多くなります。
- 起動時に ffmpeg が PATH に見つからない場合はステータスバーに警告を表示します。

DL には ffmpeg、推定には LLM エンドポイントが必要（CLI と同じ前提）。ネットワークや LLM を使わないモデル/ワーカーのオフラインテストは `uv run pytest tests/ -q` で実行できます。

## 仕組み

CLI・GUI とも処理の流れは `ファイル名 → mv2title で曲名推測（core.infer_titles）→ 形式に応じたタグへ書き込み（core.write_tags）` です。

- 推測は `mv2title.extract_titles` を使用します。`download.py` / GUI は yt-dlp から取得した**チャンネル名をアーティスト名のヒント**（`TitleInput.channel`）として渡し、推測精度を上げます。
- タグ書き込み（`core.write_title`）は形式別です: `.mp3` / `.wav` は ID3 の `TIT2` フレーム、`.m4a` は MP4 の `\xa9nam` アトム。
- 検証（推測した曲名が元タイトルに含まれるか）に失敗したファイルと空タイトルはスキップされます（GUI で手動編集した行はユーザーの意思を優先して書き込みます）。LLM 応答と対象件数が一致しない場合は、誤ったファイルにタイトルを付けるのを避けるため処理を中断します。

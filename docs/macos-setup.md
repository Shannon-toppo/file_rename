# Mac セットアップ手順書（開発初心者向け）

この手順書は、`file_rename`（GUI 版の曲名タグ付けアプリ）を **Mac で初めてセットアップする方**向けのガイドです。ターミナルの開き方から順を追って説明します。コマンドはすべてコピー&ペーストでそのまま実行できる形にしてあります。

> Windows の方はこの手順書ではなく、リポジトリの [README.md](../README.md) の手順を参照してください。

## この手順書の対象

- **Apple Silicon Mac**（M1 / M2 / M3 / M4 など）を想定しています。Intel Mac でも大筋は同じですが、Homebrew のインストール先パス（後述）が異なります。
- macOS の目安バージョン: **macOS 13 (Ventura) 以降**を推奨します（それより古くても動く可能性はありますが、新しめの方が安全です）。
- 前提知識は特に不要です。ターミナルの基本操作もこの手順書の中で説明します。

## アプリの構成（最初に理解しておくこと）

このアプリは **2 つのリポジトリ**が連携して動きます。

| リポジトリ | 公開/非公開 | URL |
|:--|:--|:--|
| `mv2title`（曲名推測ライブラリ） | 公開 | https://github.com/Shannon-toppo/mv2title |
| `file_rename`（このアプリ本体） | 非公開（プライベート） | https://github.com/Shannon-toppo/file_rename |

`file_rename` は隣にある `../mv2title` を **editable なパス依存**（同じ場所を直接参照する依存）として使います。そのため、**2 つのリポジトリを必ず同じ親フォルダの直下に並べて clone する**ことが必須です。配置を間違えると `uv sync` が `mv2title` を見つけられません。

正しい配置のイメージ:

```
~/work/            ← 親フォルダ（名前は任意）
├── mv2title/      ← 公開リポジトリ
└── file_rename/   ← このアプリ（プライベート）
```

---

## 1. ターミナルを開く

macOS には「ターミナル.app」という、コマンドを入力するためのアプリが標準で入っています。

- **開き方**: `command (⌘) + space` で Spotlight 検索を開き、「ターミナル」と入力して `return` キーを押します。
- 開いた黒い（または白い）ウィンドウに、以降のコマンドを 1 行ずつ貼り付けて `return` で実行します。
- コマンドの貼り付けは `⌘ + V`、実行結果のコピーは範囲選択してから `⌘ + C` です。

以降、「〜を実行」と書いてあるものは、このターミナルにコマンドを貼り付けて `return` を押すことを指します。

## 2. Command Line Tools と Homebrew の導入

### 2-1. Command Line Tools（開発用の基本ツール）

まず Apple 純正の開発ツール群をインストールします。次を実行してください。

```bash
xcode-select --install
```

ダイアログが出たら「インストール」を選びます（数分〜十数分かかります）。すでに入っている場合は「既にインストールされています」といった内容が表示されるので、そのまま次へ進んでかまいません。

### 2-2. Homebrew（パッケージ管理ツール）

Homebrew は Mac で各種ツールを簡単に入れるための定番ツールです。**公式サイト（https://brew.sh）**に載っている公式ワンライナーを実行します。

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

途中でパスワード（Mac のログインパスワード）の入力を求められることがあります。入力しても画面には何も表示されませんが、正しく入力されているので、そのまま `return` を押してください。

### 2-3. Apple Silicon の PATH 設定（重要）

Apple Silicon Mac では Homebrew が `/opt/homebrew` に入るため、初回はインストール直後にターミナルへ「PATH を通す」設定を促すメッセージが表示されます。その内容に従い、次の 2 行を実行してください（`zsh` を使っている前提。macOS の標準シェルは `zsh` です）。

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

> Intel Mac の場合は Homebrew が `/usr/local` に入るため、この設定は不要なことが多いです。

正しく入ったか、次で確認します。

```bash
brew --version
```

`Homebrew 4.x.x` のようにバージョンが表示されれば成功です。`command not found` が出る場合は 2-3 の PATH 設定をやり直してください。

## 3. 必要なツールのインストール（git / uv / ffmpeg）

Homebrew で 3 つのツールをまとめて入れます。

```bash
brew install git uv ffmpeg
```

- **git** … リポジトリを clone するため
- **uv** … Python の依存管理・実行ツール（このアプリの中心）
- **ffmpeg** … 音声を mp3 / wav へ変換するため

それぞれバージョンを確認します（各行を実行してバージョン番号が出れば OK です）。

```bash
git --version
uv --version
ffmpeg -version
```

## 4. 作業フォルダを作り、2 リポジトリを並べて clone

### 4-1. 親フォルダを作る

ここでは例としてホーム直下に `work` フォルダを作ります（名前は任意）。

```bash
mkdir -p ~/work
cd ~/work
```

以降のコマンドは、この `~/work` フォルダの中にいる状態で実行します。

### 4-2. 公開リポジトリ mv2title を clone

`mv2title` は公開リポジトリなので、認証なしで clone できます。

```bash
git clone https://github.com/Shannon-toppo/mv2title.git
```

### 4-3. プライベートリポジトリ file_rename を clone（要認証）

`file_rename` は**プライベートリポジトリ**のため、そのままでは clone できません。GitHub の認証が必要です。ここでは GitHub 公式 CLI（`gh`）を使う方法を推奨します（ブラウザ経由で簡単に認証できます）。

まず `gh` をインストールします。

```bash
brew install gh
```

次に GitHub にログインします。

```bash
gh auth login
```

対話形式で質問されるので、次のように選ぶのがおすすめです（矢印キーで選択、`return` で確定）。

1. `GitHub.com` を選択
2. プロトコルは `HTTPS` を選択
3. 「Authenticate Git with your GitHub credentials?」→ `Yes`
4. 認証方法は `Login with a web browser` を選択
5. 画面に表示される **one-time code**（8 桁のコード）をメモし、`return` を押すとブラウザが開くので、そこにコードを貼り付けて認証します

ログインが完了したら、`file_rename` を clone します。

```bash
gh repo clone Shannon-toppo/file_rename
```

> `gh` を使わず、素の `git clone https://github.com/Shannon-toppo/file_rename.git` でも clone できます。その場合はユーザー名と **Personal Access Token**（GitHub のパスワード代わりのトークン）の入力を求められます。初めての方は上記の `gh auth login` の方が簡単です。

### 4-4. 配置を確認

次を実行して、`mv2title` と `file_rename` の両方が並んでいることを確認します。

```bash
ls ~/work
```

`file_rename` と `mv2title` の 2 つが表示されれば正しい配置です。

## 5. LM Studio（ローカル LLM）の導入

曲名の推測には、OpenAI 互換のローカル LLM エンドポイントが必要です。ここでは **LM Studio**（Mac 対応・GUI で扱いやすい）を使う手順を説明します。

### 5-1. インストール

1. 公式サイト **https://lmstudio.ai** を開きます。
2. Mac 用のダウンロード（`.dmg`）を取得します。Apple Silicon 版が用意されているので、ネイティブ版をダウンロードしてください。
3. ダウンロードした `.dmg` を開き、LM Studio アプリを「アプリケーション」フォルダへドラッグしてインストールします。
4. 初回起動時に「開発元を確認できません」と出た場合の対処は、末尾の[トラブルシューティング](#9-トラブルシューティング)を参照してください。

### 5-2. モデルのダウンロード

LM Studio の検索/ダウンロード画面から、**gemma 系のモデル**（例: `gemma` で検索して出てくる小さめの量子化モデル）をダウンロードします。このアプリは gemma 系で開発・確認されています。

> モデルによってはダウンロードに数 GB を要します。ディスクの空き容量には余裕を持たせてください（必要量はモデルにより異なるため、あくまで目安です）。

### 5-3. ローカルサーバの起動とポート確認

LM Studio には、ダウンロードしたモデルを **OpenAI 互換のローカルサーバ**として立ち上げる機能があります。

1. LM Studio の **Developer タブ**（バージョンによっては **Local Server** などの名称）を開きます。
2. ダウンロードしたモデルを選び、サーバを **Start**（起動）します。
3. 起動すると、サーバの URL（例: `http://127.0.0.1:1234`）と**ポート番号**（既定は `1234`）が表示されます。この URL は次の `.env` 設定で使うので控えておきます。

## 6. `mv2title/.env` の作成

曲名推測ライブラリ（mv2title）は、リポジトリ直下の `.env` ファイルから接続設定を読み込みます。テンプレートは用意されていないので、自分で作成します。

ターミナルで次を実行して、`.env` を作成します（`cat` でまとめて書き込みます）。

```bash
cat > ~/work/mv2title/.env <<'EOF'
BASE_URL=http://127.0.0.1:1234/v1/
API_KEY=lm-studio
SYSTEM_PROMPT=あなたはタイトル分類の専門家です。与えられたMVのタイトルから曲名を抜き出してください。
MODEL=gemma-4-e2b-it
EOF
```

各項目の意味:

| キー | 必須 | 説明 |
|:--|:--:|:--|
| `BASE_URL` | 必須 | LM Studio のローカルサーバ URL。末尾に `/v1/` を付けます。ポートが `1234` 以外の場合はここを合わせてください。 |
| `API_KEY` | 任意 | ローカルサーバではキー検証がないため、任意の文字列で OK です（例: `lm-studio`）。 |
| `SYSTEM_PROMPT` | 任意 | LLM への指示文。上の例をそのまま使えます。最新の推奨文は [mv2title の README](https://github.com/Shannon-toppo/mv2title) を参照してください。 |
| `MODEL` | 任意 | 使用するモデル名。LM Studio で実際にロードしたモデル名に合わせると確実です（省略時の既定は `gemma-4-e2b-it`）。 |

> テキストエディット.app で作る場合は、必ず **標準テキスト形式（プレーンテキスト）**で保存し、ファイル名を `.env`（拡張子なしの `.env`）にして `mv2title` フォルダ直下へ置いてください。リッチテキストのままだと正しく読み込めません。

## 7. 依存のインストールと起動

`file_rename` フォルダへ移動して、依存をインストールします。

```bash
cd ~/work/file_rename
uv sync
```

`uv sync` は `mutagen` / `yt-dlp` / `mv2title` を含む venv（仮想環境）を作ります。初回は少し時間がかかります。

まず、ネットワークや LLM を使わない**オフラインテスト**が通るか確認します。

```bash
uv run pytest tests/ -q
```

すべて緑（`passed`）になれば、基本のセットアップは成功です。

続いて GUI を起動します。

```bash
uv run python -m gui
```

Excel 風のテーブル UI が立ち上がれば起動成功です。

## 8. 動作確認（Mac 実機チェックリスト）

以下は Mac で一通り触って確認する項目です。各項目のやり方も 1 行ずつ添えてあります。

1. **`uv run pytest tests/ -q` が全緑（オフライン）**
   … 手順 7 で実行済み。ネット・LLM 不要で全テストが `passed` になることを確認します。
2. **`uv run python -m gui` で起動、テーマ（システム/ライト/ダーク）切替**
   … ツールバーやメニューから [設定] を開き、テーマを切り替えて表示が変わることを確認します。
3. **URL 1 件で DL → 推定 → 書き込みの一気通貫（LM Studio 起動状態）**
   … 上段の URL 欄に YouTube URL を 1 件貼って [追加] → [▶ 実行] し、DL・推定・書き込みが完了（緑）になるのを確認します。
4. **Cmd+C / Cmd+V / Cmd+Z がテーブルで機能する**
   … セルを選択して `⌘ + C`、別セルで `⌘ + V`、`⌘ + Z` で元に戻ることを確認します（Mac では自動で Cmd 割当になります）。
5. **Delete（Mac の delete = Backspace）でタイトルクリア**
   … 推定タイトルのある行を選択し、`delete` キーで推定タイトルがクリアされ、再推定対象に戻ることを確認します。
6. **ログパネルに yt-dlp ログが出る（レベル「詳細」で確認）**
   … [設定] のログレベルを「詳細（すべて）」にし、[ログ] パネルを開いた状態で DL すると yt-dlp の進捗行が流れることを確認します。
7. **書き込んだ m4a/mp3 のタグを Finder の情報 / Music.app で確認**
   … 書き込み済みファイルを Finder で「情報を見る」、または Music.app に読み込んでタイトルが反映されているか確認します。

## 9. トラブルシューティング

### `ffmpeg: command not found` と出る / DL 後の変換で失敗する

ffmpeg に PATH が通っていません。次で導入と確認をします。

```bash
brew install ffmpeg
ffmpeg -version
```

`ffmpeg -version` が出ない場合は、手順 2-3 の Homebrew の PATH 設定（`eval "$(/opt/homebrew/bin/brew shellenv)"`）が済んでいるか確認してください。ターミナルを一度閉じて開き直すと反映されることがあります。

### [接続テスト] が NG になる / 推定が始まらない

LLM エンドポイントへ届いていません。次を確認してください。

- **LM Studio のローカルサーバが起動しているか**（手順 5-3 の Start 済みか）。
- **ポート番号が一致しているか**。LM Studio 側のポートと `mv2title/.env` の `BASE_URL` のポートが同じか確認します（既定は `1234`）。
- URL の末尾が `/v1/` になっているか（例: `http://127.0.0.1:1234/v1/`）。

GUI では [設定] ダイアログの [接続テスト] で疎通を確認できます。サーバ未起動のまま [▶ 実行] すると、自動で**縮退モード**（DL のみ実行して行をキューのまま残す）になります。サーバを起動してから再度 [▶ 実行] すれば、推定から続きが処理されます。

### LM Studio 初回起動で「開発元を確認できません」（Gatekeeper）

macOS のセキュリティ機能（Gatekeeper）による表示です。信頼できる公式サイトから入手したアプリであることを確認のうえ、次のいずれかで開けます。

- Finder で LM Studio アプリを **右クリック（または control + クリック）→「開く」** を選び、確認ダイアログで再度「開く」を選ぶ。
- もしくは **システム設定 → プライバシーとセキュリティ** を開き、下部に出る「"LM Studio" は開発元を確認できないため…」の横の **「このまま開く」** を押す。

一度許可すれば、次回以降は通常どおり起動できます。

### `uv sync` が mv2title を見つけられない

2 つのリポジトリの配置が正しくない可能性が高いです。`file_rename` は隣の `../mv2title` をパス依存で参照するため、**同じ親フォルダ直下に両方が並んでいる**必要があります。

```bash
ls ~/work   # file_rename と mv2title の両方が表示されるか
```

片方しか無い、または別々の場所にある場合は、手順 4 に戻って正しい配置で clone し直してください。

### 日本語ファイル名の文字化けについて

Mac では、日本語を含むファイル名の文字化けは**基本的に発生しません**（macOS の Finder・ターミナルとも UTF-8 で扱われます）。Windows 環境で見られるコンソールの文字化けを心配する必要はありません。

---

以上でセットアップは完了です。うまくいかない場合は、まず手順 8 のチェックリストのどこで止まるかを切り分け、該当するトラブルシューティングを参照してください。
